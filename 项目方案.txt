  ---
  核心创新点

  1. 智能图像分析

  用户上传图片
       ↓
  Analyzer Agent (视觉模型)
       ↓
  输出结构化报告：
  {
    "character_type": "战士",
    "facing": "right",
    "art_style": "anime_2d",
    "has_weapon": true,
    "weapon_type": "sword",
    "recommended_actions": ["attack", "idle", "run"],
    "generation_hints": {"motion_intensity": "high", ...}
  }

  2. 并行动画生成

  Orchestrator 拆分任务：
       ├─ Task 1: 待机动作 ──→ Generator-1 ──→ Result 1
       ├─ Task 2: 跑步动作 ──→ Generator-2 ──→ Result 2
       └─ Task 3: 攻击动作 ──→ Generator-3 ──→ Result 3
                                (并行执行)

  3. 质量闭环

  生成结果 → Reviewer Agent
                ↓
          ┌─────┴─────┐
          │ 评分 < 7?  │
          └─────┬─────┘
           Yes  │  No
          ↓     ↓
      自动重试  通过
      (调整参数)

  4. 跨帧一致性保证

  帧序列 → Reviewer Agent
                ↓
      检查：角色颜色/轮廓/细节是否一致？
                ↓
      不一致帧 → 标记 → 后处理修复

一、确定参与者（Actor）

  传统时序图：
  User → Frontend → Backend → Database

  Agent 架构时序图，参与者变成：

  ┌─────────────────────────────────────────────────────────┐
  │  用户层     │  应用层        │  Agent 层    │  服务层    │
  ├─────────────┼────────────────┼──────────────┼────────────┤
  │  User       │  Electron+Vue  │  Orchestrator│  Kling API │
  │             │  Frontend      │  Analyzer    │  LLM API   │
  │             │                │  Generator   │  RMBG      │
  │             │                │  Reviewer    │            │
  └─────────────┴────────────────┴──────────────┴────────────┘

  ---
  二、Mermaid 语法技巧

  Agent 架构常用的语法：

  %% 并行执行
  par 并行任务
      A -> B: 任务1
  and
      A -> C: 任务2
  end

  %% 条件分支
  alt 成功
      A -> B: 继续
  else 失败
      A -> C: 重试
  end

  %% 循环
  loop 直到成功
      A -> B: 尝试
  end

  %% 分组（视觉上区分）
  rect rgb(200, 220, 240)
      A -> B: 分析阶段
  end

  四、再画复杂版（多动作并行 + 重试）

  sequenceDiagram
      actor User as 用户
      participant FE as Vue 前端
      participant BE as FastAPI 后端
      participant ORC as Orchestrator
      participant ANA as Analyzer
      participant GEN as Generator
      participant REV as Reviewer
      participant Kling as Kling API

      User->>FE: 上传图片 + "生成待机/走路/攻击"
      FE->>BE: POST /generate

      BE->>ORC: 创建任务
      ORC->>ANA: 分析图片
      ANA-->>ORC: 分析报告 + 3个动作建议

      rect rgb(240, 240, 255)
          Note over ORC,Kling: 并行生成 3 个动作
          par 动作1: 待机
              ORC->>GEN: generate(idle)
              GEN->>Kling: 请求视频
              Kling-->>GEN: 视频
              GEN-->>ORC: idle_result
          and 动作2: 走路
              ORC->>GEN: generate(walk)
              GEN->>Kling: 请求视频
              Kling-->>GEN: 视频
              GEN-->>ORC: walk_result
          and 动作3: 攻击
              ORC->>GEN: generate(attack)
              GEN->>Kling: 请求视频
              Kling-->>GEN: 视频
              GEN-->>ORC: attack_result
          end
      end

      rect rgb(255, 245, 230)
          Note over ORC,REV: 质检阶段
          ORC->>REV: 检查 idle_result
          REV-->>ORC: 8/10 ✓

          ORC->>REV: 检查 walk_result
          REV-->>ORC: 7/10 ✓

          ORC->>REV: 检查 attack_result
          REV-->>ORC: 5/10 ✗

          loop 重试直到通过
              ORC->>GEN: 重试 attack
              GEN->>Kling: 调整参数重试
              Kling-->>GEN: 新视频
              GEN-->>ORC: attack_result_v2
              ORC->>REV: 检查
              REV-->>ORC: 7/10 ✓
          end
      end

      ORC-->>BE: 全部 PNG 序列
      BE-->>FE: 返回预览
      FE-->>User: 展示 + 导出按钮
